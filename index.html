Rebuilding the soul of a PVM

This is not a scanline filter. The monitor above runs the actual plumbing of a broadcast CRT: an NTSC encoder modulates the game onto a subcarrier, a comb filter tears luma from chroma and fails in all the correct ways, an electron beam blooms across P22 phosphor behind an aperture grille. The dot crawl is computed, not painted.

Press P for power. Then press C and watch the same game re-patched live through composite, S-Video, and RGB.

The signal chain

cartridge → encoder → comb filter → demodulator
   → deflection & beam → phosphor persistence
      → halation → grille → curved glass → you

Seven GLSL passes, each one a component you could point to on a real chassis. Degauss thunks. Power-off collapses the raster to a white dot. The 15.7 kHz flyback whine is in there too, if you can still hear it.

Built for humans and machines

Every asset is a schema-validated JSON text grid — sprites, sounds, tilemaps, even the font. No binary blobs, no registration step. Drop a file in assets/, the tube hot-reloads. A new game is one folder with a cart.ts. An AI can contribute a sprite as easily as it can write a sentence; that is the point.
